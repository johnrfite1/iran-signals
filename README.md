# Iran Signal Board

A single-page decision tracker for monitoring a phased Iran regime-change scenario across milestones, decision gates, invalidation triggers, and end-state probability estimates.

No server, no build step. Open `index.html` in a browser and start tracking.

## What It Tracks

The board models a four-phase operational timeline (D+0 through D+90):

| Phase | Window | Focus |
|-------|--------|-------|
| I — Kinetic Paralysis | Days 1–7 | Leadership decapitation, C2 severance, WMD lockdown |
| II — Artesh Pivot & Social Fracture | Days 7–14 | Military defection, clerical legitimacy withdrawal, geographic spread |
| III — Economic & Social Defibrillation | Days 15–30 | Bazaari reopening, security handover, infrastructure survival |
| IV — Transitional Consolidation | Days 30–90 | Hybrid council formation, Basij conversion, diplomatic normalization |

Each phase contains **milestones** — specific conditions tagged as Necessary (N) or Sufficient (S). You assess each milestone's status (Green / Amber / Red) and confidence level (High / Med / Low), then attach analyst notes with sourcing.

**Decision Gates** (A through D) auto-evaluate based on the milestones beneath them. A gate passes or fails according to built-in rules — you don't set gate status manually.

**Invalidation Triggers** are ten scenario-breaking events (e.g., Russia/China spoiler, humanitarian cascade, domestic political collapse). Mark a trigger as fired when its threshold is met.

**End-State Probabilities** are four modeled outcomes — Clean Transition, Negotiated Settlement, IRGC Junta, and Fragmented Conflict — set via sliders that must sum to 100%.

## How to Use

1. **Set the day counter** (D+N) at the top to reflect the current operational day. D+0 = start of operations.

2. **Work through each phase.** For every milestone:
   - Set **Status**: Green (condition met), Amber (partial/contested), Red (not met).
   - Set **Confidence**: High, Med, or Low.
   - Check the **Done** box once you've reviewed that row.
   - Optionally expand **Notes** to log evidence and source citations.

3. **Check Decision Gates.** These update automatically. A gate fails if any required [N] condition is Red, or if 2+ required [N] conditions are Amber with Low confidence.

4. **Review Invalidation Triggers.** Check "Mark trigger fired" when a threshold is clearly met. Use the notes field to log evidence.

5. **Adjust End-State Probabilities.** Drag the sliders. The board warns you if the total doesn't equal 100%, and flags when the C+D combined probability exceeds 50% (branch rule: deprioritize clean-transition planning).

6. **Write a Snapshot.** The free-text area at the top is for a current situation summary.

## Data & Persistence

All state is saved to your browser's `localStorage`. Nothing leaves your machine.

- **Export**: Downloads the full board state as a dated JSON file.
- **Import**: Merges a JSON file into the current state. Any field present in the import overwrites the current value; absent fields are preserved. This supports both full and partial updates.
- **Reset**: Clears all data back to defaults (with confirmation prompt).

## LLM-Assisted Updates

The board is designed to accept state updates generated by an LLM. The workflow:

1. Give the LLM the current situation and your assessment criteria.
2. It generates a JSON state file matching the board's schema.
3. Import the JSON via the board's Import button.

See the JSON schema and field rules in the companion LLM guide if you want to set this up.
